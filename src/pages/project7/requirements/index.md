# Requirements

## Functional requirements

### Front

La aplicación web debe tener tres páginas, una para visualizar la lista de ítems, otra para poder añadir un ítem y otra para visualizar los detalles de un ítem.

La aplicación debe gestionar los casos de error y mostrar feedback al usuario de cuando hay un problema o de cuando una acción se ha completado con éxito. También debe mostrar feedback de cuando se está cargando información.

### Back

La API REST debe tener cuatro endpoints, uno para obtener todos los ítems, otro para añadir un ítem, otro para eliminar un ítem y otro para obtener un ítem en particular.

## Technical requirements

### Front

- React
- Gestión del estado con Redux ToolKit
- Mobile only
- README con información sobre la app
- 90% coverage
- 0 bugs
- 0 duplicaciones
- 0 deuda técnica
- Lighthouse con métricas en verde explicadas
- Deploy en Netlify

### Back

- Express
- MongoDB
- README con información de uso y con endpoints (de cada endpoint método, url, body y response)
- 90% coverage
- 0 bugs
- 0 duplicaciones
- 0 deuda técnica
- Colección de endpoints de Postman exportada como JSON (en el root del proyecto)
- Deploy en Render
